Main Document

Path to main document in the project. This is your app entry point.

Can be one of the following:

    1. Path to local HTML document. E.g. /projects/myProject/index.html

    2. Web server URL (pve-proxy mode) E.g. http://myserver.com/index.html

    3. Path to JS file (node.js)

    4. Path to main HTML document (node-webkit)

Browser

This is web application. You can specify which browser to open it with. System default browser is used, if you leave it blank.

node.js

Launch node.js server or any application using node.js. You need node.js to be installed in your system. (Note. Mac-OS users will need to specify the path to node.js)

node-webkit

Launch node-webkit application. Please specify the path to node-webkit (node-webkit.app, node-webkit.exe etc) for best experience.

Console

If you are not happy with COLT launchers, you can run your custom command with any parameters. This can be used with phonegap apps.

Manual

You will start the application manually. COLT will wait for client connection when you start the session. For example, you could build phonegap app with xcode or Android Studio and upload to the device. Read more about "launching phonegap apps in live mode on the device".

Use Custom Output

COLT keeps live session files in special folder by default (ссылка на доку). You can override this with the folder of your choice (warning, do not create "out" folder inside your project folder to avoid infinite recursion)

Exclude from Livecoding Pattern

By default COLT will process all JS/HTML files, except minified scripts. You might want to exclude some files or folders from processing. For example, to improve session start time. You can specify comma-separated list of file and folder names. Paths that start with slash are interpreted as relative to project folder (e.g. /staticLibs)

Max Loop Iterations

To avoid accidental endless loops, COLT injects max iterations count checking code in your loops. Default value is 10000, longer loops will throw exception. Adjust this value if your code needs longer loops.

Don't Catch Runtime Exceptions

You can accidentally add bad code to running app during live session, that can throw runtime exceptions. To prevent this, COLT wraps every function into try/catch. If you rely on exception handling in your code, or you want error stack trace - turn this option off.

Simple Reload Mode

In some cases, e.g. if you work within framework that dynamically transforms DOM you will need to reload the page after every update. Use this option in those cases.

Clear Messages Log on Session Start

By default "Logs" tab in COLT UI retains messages from previous session (e.g., after the page was reloaded). You can turn this behavior off here.

Redirection Snippet

To launch phonegap app on the device, place this code in phonegap main HTML document, so that it will load processed code from COLT local web server.

Main Document URL

If you need the link to your app that you can send to someone, you could copy it from browser window, or, if you launch the app manually, you could get it here.
